Site Notes

About This Site

This site was built specifically for my PhD application. It's also an example of the kind of work I do: designing information architecture for specific audiences, making deliberate choices about accessibility and presentation, and shipping functional systems without unnecessary complexity.

Design

The visual language borrows from UCSC's brand identity: navy (#003C6C) and gold (#F1B521), while extending it with lighter variants for backgrounds and accents. I avoided dense bullet lists and heavy formatting in favor of prose that reads more like writing and less like a slide deck.

Typography

The primary typeface is Readex Pro, the world-script expansion of Lexend. Lexend was designed by Bonnie Shaver-Troup and Thomas Jockin specifically to improve reading fluency, and studies found it helps readers with dyslexia and reduces visual stress for everyone. Readex Pro extends that work to Arabic script (added by Nadine Chahine) while maintaining the clean, geometric letterforms that make it readable at any size.

It also renders Hawaiian diacritical marks correctly—the ʻokina and kahakō that are essential to Hawaiian language. Given my work with the Hawaiian language community, that matters to me.

Source Code Pro handles the few monospace elements I used, such as the breadcrumb navigation.

I chose Readex Pro not just for aesthetics but because accessibility research backs it. For a portfolio about technology in service of community, the typeface should embody that principle.

Accessibility

The site targets WCAG AAA compliance where possible. This includes: color contrast ratios above 7:1 for body text, skip links for keyboard navigation, semantic HTML structure, and alt text for images. I tested with WAVE and keyboard-only navigation. The gold-on-navy and navy-on-gold combinations were specifically chosen to pass contrast requirements.

Technical

Static HTML with Tailwind CSS (via CDN) and Lucide icons, hosted on GitHub Pages. No local build step, no bundler, no framework beyond utility CSS. This was intentional, because I wanted something I could edit directly and that would remain stable without maintenance or the hassles of standard web hosting. GitHub Pages seemed like the right choice, as the simplicity of it is the point.

Process

I built this site with automated assistance from Claude Code, along with other AI tools for external code review, with some direct modifications done manually. The design decisions, content, draft revisions and editorial voice are mine, but AI helped with CSS implementation, accessibility testing, and iterating through the various revisions. This felt appropriate given that my research interest is how people use AI as a collaborative tool; in doing so, this site becomes one of my design cases.
